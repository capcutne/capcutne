3. API Endpoint
POST /ai/transcribe
// Request
{
  "editorState": {
    "tracks":         [...],          // clip timing + labels
    "subtitles":      [...],          // up to 40 subtitle segments (primary source)
    "totalDuration":  62.5
  },
  "transcriptSegments": []            // optional: pass existing segments to re-use (skips AI call)
}
// Response
{
  "language":   "en",
  "transcript": [
    { "start": 0.00, "end": 4.20, "text": "..." },
    ...
  ]
}

AI Prompt design: GPT receives clip labels + subtitle text as the primary source of spoken words. It generates segment timestamps aligned to the timeline and writes natural-sounding sentences. Subtitles are used verbatim where available; unlabeled clips get inferred commentary.

6. Timeline Integration (Click → Jump)
User clicks any transcript segment row
jumpTo(time) sets playhead = time
#tl-scroll.scrollLeft repositioned to time × pxPerSec − 120 (centering the segment)
renderAll() refreshes the timeline + playhead indicator
The clicked row flashes a gold highlight for 800ms
Active segment auto-tracking: _updateActive() runs on playhead tick and highlights the currently-playing row

9. Known Limitations
Limitation	Notes
No real audio transcription	Uses GPT to infer speech from clip labels + subtitles — not actual Whisper speech-to-text
One transcript per session	Keyed as "default" — multi-asset projects (different clips) would need per-assetId generation
Transcript not auto-updated	Re-generating overwrites the previous transcript — no diff/merge
Active-segment tracking	Requires pxPerSec global — works with current capcut.html architecture
